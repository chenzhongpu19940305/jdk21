# LC0146 LRU 缓存机制（LRU Cache）

## 题目大意
设计并实现一个 LRU (Least Recently Used) 缓存，支持 `get`、`put` 操作，要求平均 O(1) 时间复杂度。

## 思路（哈希表 + 双向链表）
- 使用 **哈希表** + **双向链表**：
  - 哈希表 `key -> 节点`，O(1) 定位。
  - 双向链表维护使用顺序：
    - 头部是最近使用的，尾部是最久未使用的。
- `get(key)`：
  - 若不存在返回 -1；
  - 否则将该节点移动到链表头部。
- `put(key, value)`：
  - 若 key 已存在：更新值，并移动到头部；
  - 若不存在：新建节点加入头部；
    - 若超出容量，则删除尾部节点，并从哈希表中移除其 key。
- 使用两个哨兵节点 `head, tail` 简化插入和删除逻辑。

## 复杂度
- `get` 和 `put`：时间复杂度 O(1)。
- 空间复杂度：O(capacity)。

## 考点
- LRU 缓存的经典实现架构。
- 哈希表 + 双向链表的综合应用。
